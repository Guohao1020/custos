Custos

Nacos 原生 · 自托管 · 面向 AI Agent 的「身份·密钥·权限」统一引擎

在 GitHub 上查看 看它如何工作
Nacos AI 中心负责发现，Custos 负责安全执行 —— PEP · 密钥 · 审计
关系类比 Nacos（注册） + Sentinel（流控） —— 现在是 Nacos（AI 发现） + Custos（AI 管控）
The problem

你的 Agent 要访问数据库 / 中间件，
但你不能把真实高权限密钥交给它。

Four moats

四大护城河

每一项都已落地、可演示 —— 不是 slogan，是实证。

01

Nacos 原生

策略走 Nacos 配置 + gRPC 秒级热推，实测吊销生效 ~275ms。

02

自研引擎 + 国密

引擎 100% 自研（不抄 Vault/OpenBao 代码）；CipherSuite 国密一键切 国际/国密套件。

03

Secretless

Agent 永不持有 DB 凭证；现场签发 即用即焚 的 v_ro_*，凭证不出返回值/日志/LLM 上下文。

04

防篡改审计

每次决策落哈希链，改一行即断链定位 seq。

How it works

一次 query_db 的 secretless 路径

broker 即 PEP：校验 → 决策 → 现场签短时只读账号 → 只读执行 → 即焚 → 落审计链。全程纯服务端，凭证从不出域。

Onboard your fleet

接入你的资源 fleet 🔜 roadmap v0.5

把企业一整个 fleet 的库 / 中间件 / 系统注册进来，高权限密钥由 Custos 的 Barrier 加密托管，Agent 只拿即用即焚的短时凭证。

01 · 注册

注册资源

后台把库 / 中间件登记进资源注册表 —— 连接串、方言、角色签发模板都 config-driven。

02 · 托管

Barrier 托管高权限密钥

root/admin 凭证 Barrier 加密 落盘，绝不明文、绝不进 Nacos，仅签发时内存解出、用后 Zeroize、可轮换。

03 · 签发

Agent 拿短时凭证

Agent 经 Nacos 发现资源后，Custos 现场签发授权范围内的 即用即焚 短时凭证。
